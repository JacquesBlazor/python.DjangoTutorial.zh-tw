小部件¶

小部件是 Django 對 HTML 輸入元素的表示。該小部件處理 HTML 的呈現，以及從與該小部件相對應的 GET/POST 字典中提取資料。

內建小工具產生的 HTML 使用 HTML5 語法，目標為 . 例如，它使用布林屬性，而 不是 XHTML 樣式。<!DOCTYPE html>checkedchecked='checked'

提示

不應將小部件與表單欄位混淆。表單欄位處理輸入驗證的邏輯，並直接在範本中使用。小部件處理網頁上 HTML 表單輸入元素的呈現以及原始提交資料的提取。但是，小部件確實需要分配給表單欄位。

指定小部件¶

每當您在表單上指定欄位時，Django 將使用適合要顯示的資料類型的預設小工具。若要尋找哪個小工具用於哪個欄位，請參閱有關 內建欄位類別的檔案。

但是，如果您想要對欄位使用不同的小部件，則可以widget在欄位定義上使用參數。例如：

from django import forms


class CommentForm(forms.Form):
    name = forms.CharField()
    url = forms.URLField()
    comment = forms.CharField(widget=forms.Textarea)

這將指定一個帶有註釋的表單，該註釋使用更大的Textarea 小部件，而不是預設的TextInput小部件。

設定小部件的參數¶

許多小部件都有可選的額外參數；它們可以在欄位上定義小部件時設定。在下列範例中， years為 a 設定屬性 SelectDateWidget：

from django import forms

BIRTH_YEAR_CHOICES = ["1980", "1981", "1982"]
FAVORITE_COLORS_CHOICES = {
    "blue": "Blue",
    "green": "Green",
    "black": "Black",
}


class SimpleForm(forms.Form):
    birth_year = forms.DateField(
        widget=forms.SelectDateWidget(years=BIRTH_YEAR_CHOICES)
    )
    favorite_colors = forms.MultipleChoiceField(
        required=False,
        widget=forms.CheckboxSelectMultiple,
        choices=FAVORITE_COLORS_CHOICES,
    )

有關哪些小部件可用以及它們接受哪些參數的更多資訊，請參閱內建小部件。

繼承自 widget 的SelectWidget ¶

繼承自Selectwidget 的 widget 處理選擇。它們向用戶提供可供選擇的選項清單。不同的小部件以不同的方式呈現此選擇；小Select部件本身使用 <select>HTML 清單表示，同時RadioSelect使用單選按鈕。

Select預設情況下，在欄位上使用小部件ChoiceField。小部件上顯示的選項繼承自ChoiceField，並且更改ChoiceField.choices將更新Select.choices。例如：

>>> from django import forms
>>> CHOICES = {"1": "First", "2": "Second"}
>>> choice_field = forms.ChoiceField(widget=forms.RadioSelect, choices=CHOICES)
>>> choice_field.choices
[('1', 'First'), ('2', 'Second')]
>>> choice_field.widget.choices
[('1', 'First'), ('2', 'Second')]
>>> choice_field.widget.choices = []
>>> choice_field.choices = [("1", "First and only")]
>>> choice_field.widget.choices
[('1', 'First and only')]

自訂小部件實例¶

當 Django 將小部件呈現為 HTML 時，它僅呈現非常少的標記 - Django 不會新增類別名稱或任何其他特定於小部件的屬性。例如，這意味著所有TextInput小部件將在您的網頁上顯示相同。

有兩種自訂小部件的方法：每個小部件實例和每個小部件類別。

設定小部件實例的樣式¶

如果您想要讓一個小部件實例看起來與另一個不同，則需要在實例化小部件物件並將其指派給表單欄位時指定其他屬性（並且可能會在 CSS 檔案中新增一些規則）。

例如，採用以下形式：

from django import forms


class CommentForm(forms.Form):
    name = forms.CharField()
    url = forms.URLField()
    comment = forms.CharField()

該表單將包括三個預設TextInput小部件，具有預設渲染 - 沒有 CSS 類，沒有額外的屬性。這意味著為每個小部件提供的輸入框將呈現完全相同：

>>> f = CommentForm(auto_id=False)
>>> print(f)
<div>Name:<input type="text" name="name" required></div>
<div>Url:<input type="url" name="url" required></div>
<div>Comment:<input type="text" name="comment" required></div>

在真實的網頁上，您可能不希望每個小部件看起來都一樣。您可能需要更大的註釋輸入元素，並且您可能希望“名稱”小部件具有一些特殊的 CSS 類別。也可以指定“type”屬性以利用新的 HTML5 輸入類型。為此，您可以Widget.attrs在建立小部件時使用參數：

class CommentForm(forms.Form):
    name = forms.CharField(widget=forms.TextInput(attrs={"class": "special"}))
    url = forms.URLField()
    comment = forms.CharField(widget=forms.TextInput(attrs={"size": "40"}))

您也可以修改表單定義中的小工具：

class CommentForm(forms.Form):
    name = forms.CharField()
    url = forms.URLField()
    comment = forms.CharField()

    name.widget.attrs.update({"class": "special"})
    comment.widget.attrs.update(size="40")

或者，如果該欄位未直接在表單上聲明（例如模型表單欄位），則可以使用該Form.fields屬性：

class CommentForm(forms.ModelForm):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.fields["name"].widget.attrs.update({"class": "special"})
        self.fields["comment"].widget.attrs.update(size="40")

然後 Django 將在渲染的輸出中包含額外的屬性：

>>> f = CommentForm(auto_id=False)
>>> print(f)
<div>Name:<input type="text" name="name" class="special" required></div>
<div>Url:<input type="url" name="url" required></div>
<div>Comment:<input type="text" name="comment" size="40" required></div>

id您也可以使用設定 HTML attrs。請 BoundField.id_for_label參閱範例。

以及一種必需的方法：

decompress（價值）¶

此方法從欄位中取得單一「壓縮」值並傳回「解壓縮」值清單。可以假定輸入值有效，但不一定非空。

此方法必須由子類別實現，並且由於值可能為空，因此實現必須是防禦性的。

「解壓縮」背後的基本原理是，有必要將表單欄位的組合值「拆分」為每個小部件的值。

一個範例是如何將值SplitDateTimeWidget轉換 datetime為列表，其中日期和時間分為兩個單獨的值：

from django.forms import MultiWidget


class SplitDateTimeWidget(MultiWidget):
    # ...

    def decompress(self, value):
        if value:
            return [value.date(), value.time()]
        return [None, None]

下面是一個範例小部件，它的子類別MultiWidget用於在不同的選擇框中顯示帶有日、月和年的日期。DateField這個小部件旨在與 a而不是 a一起使用MultiValueField，因此我們實現了 value_from_datadict()：

from datetime import date
from django import forms


class DateSelectorWidget(forms.MultiWidget):
    def __init__(self, attrs=None):
        days = {day: day for day in range(1, 32)}
        months = {month: month for month in range(1, 13)}
        years = {year: year for year in [2018, 2019, 2020]}
        widgets = [
            forms.Select(attrs=attrs, choices=days),
            forms.Select(attrs=attrs, choices=months),
            forms.Select(attrs=attrs, choices=years),
        ]
        super().__init__(widgets, attrs)

    def decompress(self, value):
        if isinstance(value, date):
            return [value.day, value.month, value.year]
        elif isinstance(value, str):
            year, month, day = value.split("-")
            return [day, month, year]
        return [None, None, None]

    def value_from_datadict(self, data, files, name):
        day, month, year = super().value_from_datadict(data, files, name)
        # DateField expects a single string that it can parse into a date.
        return "{}-{}-{}".format(year, month, day)

建構Select函式在清單中建立多個小部件。此 super()方法使用此列表來設定小部件。

所需的方法將值decompress()分解 datetime.date為與每個小部件對應的日、月和年值。如果選擇了無效日期，例如不存在的 2 月 30 日，則會DateField向此方法傳遞字串，因此需要解析。Finalreturn處理何時 valueis None，這意味著我們的子小部件沒有任何預設值。

預設實作會傳value_from_datadict()回與每個 對應的值列表Widget。當 aMultiWidget與 a 一起使用時，這是合適的MultiValueField。但由於我們想將此小部件與 a 一起使用DateField，它採用單一值，因此我們重寫了此方法。這裡的實作將子小部件中的資料組合成預期格式的字串DateField。

<fieldset>
    <legend>{{ myform.beatles.label }}</legend>
    {% for radio in myform.beatles %}
    <div class="myradio">
        {{ radio }}
    </div>
    {% endfor %}
</fieldset>

這將產生以下 HTML：

<fieldset>
    <legend>Radio buttons</legend>
    <div class="myradio">
        <label for="id_beatles_0"><input id="id_beatles_0" name="beatles" type="radio" value="john" required> John</label>
    </div>
    <div class="myradio">
        <label for="id_beatles_1"><input id="id_beatles_1" name="beatles" type="radio" value="paul" required> Paul</label>
    </div>
    <div class="myradio">
        <label for="id_beatles_2"><input id="id_beatles_2" name="beatles" type="radio" value="george" required> George</label>
    </div>
    <div class="myradio">
        <label for="id_beatles_3"><input id="id_beatles_3" name="beatles" type="radio" value="ringo" required> Ringo</label>
    </div>
</fieldset>

其中包括<label>標籤。要獲得更精細的資訊，您可以使用每個單選按鈕的tag,choice_label和id_for_label屬性。例如，這個範本...

<fieldset>
    <legend>{{ myform.beatles.label }}</legend>
    {% for radio in myform.beatles %}
    <label for="{{ radio.id_for_label }}">
        {{ radio.choice_label }}
        <span class="radio">{{ radio.tag }}</span>
    </label>
    {% endfor %}
</fieldset>

...將產生以下 HTML：

<fieldset>
    <legend>Radio buttons</legend>
    <label for="id_beatles_0">
        John
        <span class="radio"><input id="id_beatles_0" name="beatles" type="radio" value="john" required></span>
    </label>
    <label for="id_beatles_1">
        Paul
        <span class="radio"><input id="id_beatles_1" name="beatles" type="radio" value="paul" required></span>
    </label>
    <label for="id_beatles_2">
        George
        <span class="radio"><input id="id_beatles_2" name="beatles" type="radio" value="george" required></span>
    </label>
    <label for="id_beatles_3">
        Ringo
        <span class="radio"><input id="id_beatles_3" name="beatles" type="radio" value="ringo" required></span>
    </label>
</fieldset>

如果您決定不循環單選按鈕（例如，如果您的範本包含），它們將在with 標籤中輸出，如上所述。{{ myform.beatles }}<div><div>

外部<div>容器接收id小部件的屬性（如果已定義）或BoundField.auto_id其他屬性。

當循環單選按鈕時，label和input標籤分別包含 for和id屬性。每個單選按鈕都有一個 id_for_label屬性來輸出元素的 ID。

CheckboxSelectMultiple¶

班級 CheckboxSelectMultiple¶
  • template_name:'django/forms/widgets/checkbox_select.html'
  • option_template_name:'django/forms/widgets/checkbox_option.html'

與 類似SelectMultiple，但呈現為複選框清單：

<div>
  <div><input type="checkbox" name="..." ></div>
  ...
</div>

外部<div>容器接收id小部件的屬性（如果已定義）或BoundField.auto_id其他屬性。

與 一樣RadioSelect，您可以循環遍歷小工具選擇的各個複選框。與 不同的是RadioSelect，如果該欄位是必需的，則複選框不會包含 requiredHTML 屬性，因為瀏覽器驗證將要求選取所有複選框而不是至少一個。

當循環遍歷複選框時，label和input標籤分別包含 for和id屬性。每個複選框都有一個 id_for_label屬性來輸出元素的 ID。

一個小部件，用於處理具有值清單的欄位的多個隱藏小部件。

SplitDateTimeWidget¶

班級 SplitDateTimeWidget¶
  • template_name:'django/forms/widgets/splitdatetime.html'

包裝（使用MultiWidget）兩個小部件：DateInput 日期和TimeInput時間。必須與 一起使用 SplitDateTimeField而不是DateTimeField.

SplitDateTimeWidget有幾個可選參數：

date_format¶

如同DateInput.format

time_format¶

如同TimeInput.format

date_attrs¶
time_attrs¶

如同Widget.attrs。DateInput包含分別在渲染和 小部件上設定的 HTML 屬性的字典TimeInput。如果未設定這些屬性，Widget.attrs則使用 。

SplitHiddenDateTimeWidget¶

班級 SplitHiddenDateTimeWidget¶
  • template_name:'django/forms/widgets/splithiddendatetime.html'

與 類似SplitDateTimeWidget，但HiddenInput同時用於日期和時間。

SelectDateWidget¶

班級 SelectDateWidget¶
  • template_name:'django/forms/widgets/select_date.html'

包裝三個Select小零件：每個月、日和年。

採用幾個可選參數：

years¶

在「年份」選擇框中使用的選用年份清單/元組。預設值是包含當前年份和未來 9 年的清單。

months¶

在“月份”選擇框中使用的可選月份字典。

字典的鍵對應於月份編號（1-索引），值是顯示的月份：

MONTHS = {
    1: _("jan"),
    2: _("feb"),
    3: _("mar"),
    4: _("apr"),
    5: _("may"),
    6: _("jun"),
    7: _("jul"),
    8: _("aug"),
    9: _("sep"),
    10: _("oct"),
    11: _("nov"),
    12: _("dec"),
}
empty_label¶

如果DateField不需要， SelectDateWidget則清單頂部將有一個空選項（---預設）。您可以使用該屬性來變更此標籤的文字empty_label。empty_label可以是 string、list、 或tuple。使用字串時，所有選擇框都會有一個帶有此標籤的空選項。如果empty_label 是 3 個字串元素的一個list或tuple多個，則選擇框將有自己的自訂標籤。標籤應按此順序排列 。('year_label', 'month_label', 'day_label')

# A custom empty label with string
field1 = forms.DateField(widget=SelectDateWidget(empty_label="Nothing"))

# A custom empty label with tuple
field1 = forms.DateField(
    widget=SelectDateWidget(
        empty_label=("Choose Year", "Choose Month", "Choose Day"),
    ),
)
